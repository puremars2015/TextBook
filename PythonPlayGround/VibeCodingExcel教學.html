Why

為什麼是 Office Scripts？

優點（適合 Excel 自動化）

  • 不用安裝 Python / 套件相依問題少
  • 直接跑在 Excel（多為網頁版）
  • 可搭配 Power Automate：排程、批次處理、通知
✓
最適用情境
表格清洗、欄位標準化、彙總報表、Top10 排名、套用範本格式

限制（務必先知道）

  • 主要在 Excel 網頁版編寫與執行
  • 多為操作「當前活頁簿」
  • 批次多檔通常要靠 Power Automate 迴圈呼叫
  • 不等同本機程式：不能自由讀寫本機檔案路徑
!
檔案建議位置
OneDrive / SharePoint（更好自動化、排程、觸發）

Core Idea

Vibe Coding 的核心觀念

不是「叫 AI 幫你寫腳本」，而是：
把需求寫成可驗證的規格，讓 AI 迭代產出直到符合規格。

規格化

講清楚：Sheet 名稱、標題列、欄位名、輸出位置、覆蓋/新增、驗證方式。

迭代

先 MVP（讀→驗證→寫），每次只加一個規則，靠錯誤訊息與事實回饋修正。

Workflow

Vibe Coding 操作 Excel 的整體流程（Office Scripts 版）

步驟 你要做的事 寫給 AI 的關鍵資訊
1. 定義 I/O 資料在哪裡、要輸出到哪裡 Sheet / Range 或 Table / 欄位名 / 覆蓋或新建
2. 先做 MVP 最小可行：讀→驗證→寫 getUsedRange、檢查欄位、複製到新 sheet
3. 小步迭代 一次只加一個規則 過濾 → 標準化 → 彙總 → 排序 → Top10
4. 回饋修正 用事實回饋，不要只說「不對」 錯誤訊息、實際欄位名、前 10 列樣本（可遮敏）
i
固定骨架
要求回傳「可直接貼進 Script Editor 的完整腳本」，避免片段造成拼裝錯誤。

Setup

環境準備與基本設定（Office Scripts）

你需要的東西

  • Microsoft 365 帳號（可用 Excel 網頁版）
  • 檔案建議放 OneDrive / SharePoint

如何開啟 Script Editor

  1. 用瀏覽器開 Excel（for the web）
  2. 功能區：Automate（自動化）
  3. New Script（新指令碼） → 貼上腳本
  4. Run（執行）

資料模型（一定要懂）

  • Worksheet：工作表
  • Range：一塊儲存格區域（例如 getUsedRange()）
  • values：getValues() 回傳二維陣列
✓
建議習慣
用「欄位名稱找索引」而不是寫死 A/B/C；一次讀→記憶體處理→一次寫回。
!
看不到 Automate？
多半是授權或系統管理員停用；也可能你開的是桌面版不是網頁版。

Specification

需求描述技巧（核心章節）

需求描述模板（必填資訊）

  • 活頁簿狀態：網頁版執行、資料在哪個 sheet、標題列位置
  • 欄位定義：用「欄位名」而非 A/B/C
  • 處理規則：每條規則可驗證（例如「移除金額 0」）
  • 輸出結果：寫入哪個 sheet、是否覆蓋、格式需求
  • 限制偏好：完整腳本、錯誤提示、避免逐格讀寫
  • 驗證方式：Top10、列數一致、欄位存在

讓 AI 更穩的三個要求

1
先確認資料長相
標題列是否第 1 列？欄位名是否有空白？日期是文字還是 Excel 序號？
2
先做 MVP 再加規則
先把 Data 複製到 CleanData，確保腳本可跑，再逐條加規則。
3
固定用欄位名找索引
避免換欄位順序就壞掉；用 headers → indexOf 的方式定位欄位。

Prompt

可直接貼給 AI 的 MVP Prompt

目標：先讓腳本跑起來（讀→驗證→寫），再迭代加規則。

請你用 Office Scripts（TypeScript）寫腳本，不要用 Python。 先做 MVP： 1) 讀取 Data 工作表的 usedRange 2) 印出欄位名稱（用 throw Error 顯示也可以）或至少驗證欄位存在 3) 把整份資料複製到 CleanData 工作表 完成後再問我：要加哪些清洗/彙總規則。
✓
關鍵習慣
要求回傳「完整可貼上的 Office Script」，包含 function main(workbook: ExcelScript.Workbook) 與錯誤提示。

Example

基礎範例：一句話 → 規格 → 可執行

不夠清楚的需求

「幫我把銷售資料整理成報表。」
×
問題
缺少 sheet/欄位/輸出位置/驗證方式，AI 只能猜，容易產出跑不動的腳本。

可執行規格（建議）

  • Data：第 1 列是標題
  • 移除「金額」空或 0
  • 日期標準化：Excel 序號/文字 → yyyy-mm-dd
  • 依「業務」彙總：總金額、筆數，排序
  • 輸出：CleanData（明細）與 Summary（彙總）覆蓋
✓
驗證
Summary 顯示 Top10；CleanData 列數等於符合規則列數。

Debug

常見錯誤與除錯方式（Office Scripts）

最常見錯誤

×
Range.setValues: The argument is invalid
通常是「二維陣列每列欄數不一致」。寫回前先補齊每列欄數。
!
getUsedRange() 是空的
工作表真的沒資料或資料在別的 sheet；必要時改用 Table 方式讀取。
!
getWorksheet() 回傳 undefined
sheet 名稱拼錯（含空白/全形空格）或不存在；建議複製貼上名稱。

效能/逾時

  • 避免逐格讀寫、逐列 getRange()
  • 一次讀取 → 記憶體處理 → 一次寫回
  • 多檔批次：交給 Power Automate 迴圈逐檔跑腳本
✓
回饋 AI 的正確方式
提供錯誤訊息、欄位名列表、前 10 列樣本（可遮敏），並說明「只改哪一段」。

Automation

進階：Office Scripts + Power Automate（何時需要？）

需要 Power Automate 的情境

  • 每天固定時間跑（排程）
  • 資料夾多檔批次清洗/彙總
  • 腳本跑完要後續動作：寄信/Teams 通知/搬檔/轉 PDF
→
典型流程
列出資料夾檔案 → for each 檔案 → Run script → 後續通知或歸檔。

實務案例（可擴充）

  • 案例 A：合併 Jan/Feb/Mar → AllMonths，新增「月份」欄
  • 案例 B：KPI Top10（依部門/業務彙總：總金額、平均、成交數）
  • 案例 C：欄位清洗標準化（去空白、欄位同義合併、金額去逗號轉數值）
✓
寫給 AI 的重點
欄位名、標題列位置、輸出 sheet、是否覆蓋、以及驗證方式（Top10、列數一致）。

SOP

建立自己的 Vibe Coding 工作流程（SOP）

固定 SOP（建議直接照做）

  1. 講清楚 Excel 結構：sheet / 標題列 / 欄位名 / 輸出位置
  2. 先做 MVP：讀 usedRange → 檢查欄位 → 寫到新 sheet
  3. 小步迭代：一次只加一個規則（過濾→標準化→彙總→格式）
  4. 保存成功版本：prompt、腳本版本、錯誤與解法（可複用）

下一步（選一個做就好）

1
把你的任務改寫成模板
使用「目標/欄位/規則/輸出/驗證」五段式。
2
先做 MVP（讀→寫）
先跑通再加規則，避免一次生成一大段全壞。
3
再加規則（清洗/彙總/Top10）
每次只改一段，回饋要用事實（錯誤/欄位/樣本）。
你提供：sheet 名稱、欄位名、想要的輸出樣子
就能快速客製成你的「真實版本」腳本。
