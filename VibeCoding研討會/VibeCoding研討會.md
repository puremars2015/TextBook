# Vibe Coding研討會
- AI 時代的「帶領 AI 寫程式」能力

## 主題 
- AI不只是工具，而是你的數位員工
- 人負責思考與決策大方向，AI負責執行
- Vibe Coding不是一個技術，是一個命令AI做事的概念

## 甚麼是Vibe Coding
- 狹義:用類似Lovable、Replit、VSCode這樣的產品，透過對話驅動AI來幫我們把程式做出來
- 廣義:用對話去驅動AI Agent來幫我們做事

## 甚麼是AI Agent
- 定義:可以幫你做事的 AI，而不是只有對話

## OpenClaw的爆紅
- 2026年初，一個名為OpenClaw的開源軟體突然爆紅，成為了網路上的熱門話題，甚至引起了媒體的關注。
- OpenClaw的功能包括AI的思考跟推理能力、console的操作能力、telegram的訊息傳遞能力、網路爬蟲的能力、鼠標鍵盤的操作能力以及畫面的辨識能力。
- 這些能力的結合，讓ChatGPT不再只是個聊天機器人，而是成為了一個可以跟人一樣做事的數位員工。

## 為什麼我們今天沒教你OpenClaw
- OpenClaw的功能雖然強大，但他就是一把削鐵如泥的匕首，還沒有裝刀柄，功力不夠高深的人，拿到這把匕首可能會傷到自己。
- OpenClaw所需的上下文長度問題尚未解決，當反覆交談之後，AI可能會忘記之前的對話內容，導致行動跟你期望的不一致。
- 如果你想試試看OpenClaw，建議先在虛擬機裡面試，或者在不重要的電腦上面試，避免對你的生活造成損害。

## Vibe Coding當今的缺陷
1. 生成結果不穩定
2. 看起來合理但其實錯誤的程式碼
 - 需要人工檢查與修正
3. 大型系統維護困難
 - 生成的程式碼可能缺乏結構與可維護性
4. Context Length限制
 - 無法處理過長的程式碼或複雜的系統需求，當反覆交談之後，AI可能會忘記之前的對話內容，導致生成的程式碼不連貫或不完整。
5. 安全性問題
 - 生成的程式碼可能存在安全漏洞，尤其是當AI沒有足夠的上下文來理解安全最佳實踐時。

## 進入Vibe Coding
1. 環境的選擇
- 我們選擇vscode，因為它是目前最流行的程式碼編輯器，並且有很多插件可以幫助我們與AI互動。
2. vscode的優勢
- 預裝copilot，讓我們可以直接使用AI來幫助我們寫程式碼。
- 支援多種程式語言，適合不同的開發需求。
- 可以選擇不同的LLM供應商，讓我們可以根據需求選擇最適合的AI模型。
3. LLM供應商的選擇
- 我們這邊選擇OpenRouter，因為它提供了多種LLM模型的接入，讓我們可以靈活地選擇適合我們需求的AI模型。
- 免費的API金鑰，讓我們可以輕鬆地開始使用AI來幫助我們寫程式碼。

- Python 3.10以上版本，因為很多AI相關的庫都需要Python 3.10以上的版本才能正常運行。
