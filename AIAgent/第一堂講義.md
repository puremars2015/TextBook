# AI Agent 入門與實務講義

**主題：從概念到實作（以 OpenClaw 為例）**

---

## 一、什麼是 AI Agent？

### 1. 定義

AI Agent（人工智慧代理）是一種能夠：

* 理解目標（Goal）
* 自主規劃（Planning）
* 使用工具（Tool Use）
* 持續迭代（Iteration）

來完成任務的系統。

👉 核心概念：

```
輸入目標 → 拆解任務 → 呼叫工具 → 回饋結果 → 持續優化
```

---

### 2. 與傳統 AI 的差異

| 類型    | 傳統 AI（ChatGPT） | AI Agent |
| ----- | -------------- | -------- |
| 回應方式  | 單次回覆           | 多步驟行動    |
| 是否會行動 | 不會             | 會        |
| 工具使用  | 被動             | 主動       |
| 任務完成  | 使用者主導          | AI 主導    |

---

## 二、AI Agent 的核心組成

### 1. LLM（大語言模型）

負責：

* 理解語意
* 產生計畫
* 決策下一步

---

### 2. Tool（工具系統）

Agent 可以呼叫外部能力，例如：

* CLI 指令
* API
* 資料庫
* 檔案操作

---

### 3. Memory（記憶）

* 短期記憶（上下文）
* 長期記憶（知識庫）

---

### 4. Planning（規劃能力）

範例：

```
目標：幫我產生報表

Step1: 讀取資料
Step2: 清理資料
Step3: 分析
Step4: 輸出 Excel
```

---

## 三、AI Agent 運作流程

```
使用者需求
    ↓
LLM 理解
    ↓
任務拆解（Planning）
    ↓
呼叫工具（Tool Execution）
    ↓
結果回饋（Observation）
    ↓
再次決策（Iteration）
    ↓
完成任務
```

## 五、各產業應用說明與導覽

### 1. 為什麼要先看產業應用？

因為 AI Agent 不只是聊天工具，而是可以嵌入「既有工作流程」中，幫忙：

* 接資料
* 做判斷
* 呼叫工具
* 產出結果

所以在導入時，建議先問三件事：

1. 這個產業每天重複做什麼事？
2. 哪些流程需要查資料、整理資料、回報結果？
3. 哪些步驟可以交給 Agent 半自動或全自動完成？

---

### 2. 各產業應用地圖

| 產業 | 常見需求 | AI Agent 可做的事 | 適合搭配的工具 |
| --- | --- | --- | --- |
| 製造業 | 報工、異常回報、MES/ERP 查詢 | 自動整理工單、彙整設備異常、產出日報表 | Excel、資料庫、MES API、Email |
| 零售電商 | 訂單查詢、客服回覆、庫存追蹤 | 自動回覆常見問題、整理出貨清單、追蹤缺貨品項 | CRM、Google Sheet、Webhook、LINE |
| 金融保險 | 對帳、審核、文件整理 | 彙整交易資料、比對異常、生成稽核摘要 | 資料庫、PDF、Excel、內部簽核系統 |
| 醫療行政 | 掛號、表單、衛教資訊整理 | 協助病患問答、整理文件、產出行政報表 | 表單系統、文件庫、Chat UI |
| 教育訓練 | 課程問答、作業整理、教材生成 | 自動整理講義、建立測驗題、回覆學員常見問題 | LMS、文件檔、簡報工具 |
| 行銷營運 | 名單整理、內容生成、活動追蹤 | 蒐集資料、生成文案初稿、整理活動成效 | Google Sheet、社群 API、CRM |

### 3. 課堂導覽方式（你可直接這樣帶）

你可以用以下順序帶學員理解：

#### (1) 先看「工作流程」

不要先講模型多厲害，先問：

```
這個產業每天有哪些固定流程？
哪些流程最花時間？
哪些流程最容易出錯？
```

#### (2) 再看「Agent 能接手哪一段」

例如：

```
接收需求 → 查資料 → 整理內容 → 產出報表/回覆
```

這樣學員會更容易理解 Agent 不是取代所有人，而是先取代重複工作。

#### (3) 最後看「需要哪些工具串接」

例如：

* 要查資料，就接資料庫或 API
* 要出報表，就接 Excel 或 Google Sheet
* 要做通知，就接 LINE、Email 或 Slack

---

### 4. 各產業講解範例口條

#### 製造業

「如果今天產線主管每天都要整理報工資料、查異常紀錄、彙整成 Excel 日報，這其實就很適合用 AI Agent。因為它可以先去抓資料，再做整理，最後自動輸出報表。」

#### 零售電商

「如果客服每天都在回答訂單查詢、退款進度、出貨時間，Agent 就可以先接住第一層問題，把標準化問題自動回覆，再把特殊案件轉給真人。」

#### 金融保險

「如果每天要做大量對帳、文件比對、異常標記，Agent 可以協助先把資料整理好，但最終決策仍要保留審核與稽核機制。」

---

### 5. 導入產業場景時要提醒學員的事

* 不要一開始就做全自動，先從單一流程切入
* 有個資、金流、醫療資料時，要特別注意權限與稽核
* Agent 最適合先做「資料整理、流程銜接、報表產出」
* 高風險決策仍應保留人工確認

## 六、如何用 AI 增加個人工作生產力

### 1. 先建立正確觀念

很多人以為 AI 的用途只是「幫我回答問題」，但在工作上更有價值的做法是：

* 幫我整理資訊
* 幫我產生初稿
* 幫我檢查遺漏
* 幫我減少重複操作

也就是說，AI 最適合先做你的「副手」，而不是一開始就當成完全獨立的決策者。

---

### 2. 個人生產力的四個切入點

#### (1) 資料整理

適合情境：

* 會議記錄整理
* Excel 欄位清理
* 文件摘要
* 郵件重點整理

範例：

```
請把以下會議逐字稿整理成：
1. 決議事項
2. 待辦事項
3. 負責人
4. 截止日
```

#### (2) 內容產出

適合情境：

* Email 初稿
* 提案大綱
* 簡報架構
* 教學講義初稿

範例：

```
請幫我把這份專案資訊整理成一封專業 Email，
對象是主管，語氣精簡，重點放在進度、風險、下一步。
```

#### (3) 決策輔助

適合情境：

* 比較方案差異
* 彙整優缺點
* 建立檢查清單
* 找出風險點

範例：

```
請比較 A 方案與 B 方案的差異，
分成成本、時間、風險、維護性四個面向。
```

#### (4) 流程自動化

適合情境：

* 固定格式報表
* 例行資料彙整
* 表單資料轉換
* 定期通知

範例：

```
每天把表單資料匯出後，
自動整理成 Excel，並寄給指定主管。
```

---

### 3. 工作上可直接使用的例子

#### 例子 1：上班族的 Email 生產力

情境：每天都要回覆大量郵件。

AI 可以幫你：

* 先摘要郵件重點
* 生成回信初稿
* 依不同對象調整語氣

你在課堂上可以這樣說：

「以前回 10 封信可能要花 30 分鐘，現在可以先讓 AI 產生初稿，人只要做最後確認與修正。」

#### 例子 2：主管的會議整理生產力

情境：會議結束後，要整理紀錄與待辦。

AI 可以幫你：

* 將逐字稿整理成重點摘要
* 抓出待辦事項
* 指派負責人與時間點

你在課堂上可以這樣說：

「會議真正花時間的，不是開會本身，而是會後整理。這一段非常適合交給 AI 先做第一版。」

#### 例子 3：行政人員的報表生產力

情境：每週都要做固定格式報表。

AI 可以幫你：

* 讀取原始資料
* 清理欄位格式
* 產出週報或月報

你在課堂上可以這樣說：

「如果每週都在做相同格式的整理，那就不是人的價值所在，應該優先讓 AI 或 Agent 接手。」

#### 例子 4：講師或顧問的教材生產力

情境：需要快速整理課程內容。

AI 可以幫你：

* 將資料整理成章節大綱
* 生成案例與練習題
* 協助改寫成不同程度版本

你在課堂上可以這樣說：

「同一份內容，可以請 AI 幫你改寫成初學版、主管版、技術版，這會大幅減少備課時間。」

---

### 4. 教學時可以強調的觀念

#### 不是省掉工作，而是省掉低價值重複工作

AI 最有價值的地方，不是完全取代人，而是把人從重複、瑣碎、可規則化的工作中解放出來。

#### 先求加速，再求自動化

教學時可提醒學員：

```
第一階段：用 AI 幫你更快完成
第二階段：把固定流程半自動化
第三階段：再考慮 Agent 全流程接手
```

#### 人仍然負責最後判斷

尤其在對外回覆、財務、法務、醫療、合約等場景，AI 可以先起草，但最後確認權仍應在人。

## 八、風險與限制（重點）

### 1. CLI 安全問題

AI Agent 可能會：

* 刪除檔案（rm -rf）
* 執行危險指令

👉 解法：

* CLI 白名單（Allowlist）
* Sandbox 環境
* 權限隔離

---

### 2. 幻覺（Hallucination）

* 可能生成錯誤指令
* API 調用錯誤

---

### 3. 成本問題

* Token 消耗高
* 多步驟 = 多成本

---
