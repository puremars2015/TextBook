# 第14堂｜案例分析：敏捷管理自動化—智慧週報生成系統

## 學習目標

完成本堂課後，學員將能夠：

- 了解敏捷（Agile）專案管理的基本概念與資料結構
- 整合來自多個來源的跨專案工作資料
- 使用 AI 從原始工作紀錄中自動提取關鍵里程碑
- 將原始工作資料轉化為具備管理深度的 PPT 決策報表

---

## 核心概念說明

### 1. 敏捷管理的週報痛點

**典型情境：**

PM（專案經理）每週五要整理週報，流程通常是：

1. 從 Jira 匯出本週的任務清單（CSV）
2. 打開 Excel 手動計算完成率、延遲任務數、風險項目
3. 詢問各組組長本週的主要成果
4. 整理以上資訊，寫成 Word 報告或 PowerPoint
5. 每週重複這個 2-3 小時的工作

**企業的實際痛點：**

| 問題 | 影響 |
|------|------|
| 資料散落各處 | 需要從多個系統蒐集，費時且易遺漏 |
| 格式不一致 | 不同組別的周報格式不同，彙整困難 |
| 分析深度不足 | 時間緊迫下，大多是數字羅列，缺乏洞察 |
| 版本管理問題 | 找上週的報告耗時 |
| 高重複性工作 | PM 寶貴的時間被這類工作佔據 |

---

### 2. 敏捷管理的核心資料結構

要自動化週報生成，需要理解敏捷管理的基本資料：

**Sprint（衝刺）：**
- 敏捷開發的基本工作週期，通常 1-2 週
- 每個 Sprint 有明確的開始和結束日期
- Sprint 結束後要達成一個「可交付的成果」

**Story / Task：**
- Sprint 中的工作任務
- 每個任務有：標題、負責人、狀態、預估工時、實際工時、優先級

**里程碑（Milestone）：**
- 專案的重要轉折點或交付節點
- 不是每週都有，但需要在報告中特別標注

**KPI 指標：**

| 指標 | 計算方式 | 說明 |
|------|---------|------|
| Sprint 完成率 | 完成任務數 / 計劃任務數 | 衡量計畫執行能力 |
| 工時燃盡率 | 已用工時 / 計畫工時 | 判斷是否超支 |
| Bug 密度 | 發現的 Bug 數 / 完成的功能數 | 衡量品質 |
| 阻塞任務比例 | 阻塞中任務數 / 總任務數 | 識別風險 |

---

### 3. 跨專案資料整合架構

當公司有多個並行專案時，週報需要呈現所有專案的整合視圖：

**資料來源可能包含：**

```
資料來源 1：Jira / Trello / Asana
    └── 任務清單（CSV 或 API）

資料來源 2：Git（GitHub / GitLab）
    └── Commit 記錄（代碼提交歷史）

資料來源 3：日常工作紀錄
    └── 會議記錄（Word / Notion）
    └── 組員的週記（純文字）

資料來源 4：時程表
    └── 原始計畫甘特圖（Excel）
```

**整合挑戰：**

- 不同系統的任務 ID 不相同
- 同一個任務在不同系統中可能有不同的名稱
- 時區不一致
- 部分系統可能暫時無法連接

---

### 4. AI 提取里程碑的邏輯

原始工作紀錄通常是這樣的文字：

> 「本週完成了登入模組的單元測試，所有 32 個測試案例通過。前端 UI 設計稿已獲客戶確認，可以開始實作。API 文件初版完成，已送審。」

AI 需要從這段文字中識別：
- **里程碑事件**：「所有 32 個測試案例通過」→ 測試完成里程碑
- **待追蹤事項**：「已送審」→ 有待確認的項目
- **相依關係**：「已獲客戶確認，可以開始實作」→ 解鎖了下一個任務

**AI 識別里程碑的 Prompt 設計：**

```
你是一位資深的 PM，請從以下工作紀錄中識別本週的關鍵里程碑。
里程碑的定義：
1. 重要功能完成並通過驗收
2. 關鍵決策已獲批准
3. 重要的外部依賴已解決
4. 階段性交付物完成

對於每個里程碑，請說明：
- 里程碑名稱（簡潔的標題）
- 達成日期
- 影響（對專案的意義）
- 下一步行動

工作紀錄：[內容]
```

### 5. PPT 決策報表設計原則

面向管理層的 PPT 報告，與面向執行層的工作報告截然不同：

**管理層關注的問題：**
- 我們本週有沒有達成目標？
- 哪裡有風險需要我關注？
- 有沒有需要我做決定的事項？

**PPT 簡報頁面規劃：**

| 頁碼 | 頁面標題 | 內容 |
|-----|---------|------|
| 1 | 封面 | 週報標題、週次、報告日期 |
| 2 | 本週整體健康度 | 交通燈狀態（紅/黃/綠）+ 一句話摘要 |
| 3 | 關鍵里程碑 | 本週達成 + 下週預計 |
| 4 | 各專案 KPI 儀表板 | 完成率、工時燃盡、品質指標的視覺化 |
| 5 | 風險與阻塞項目 | 需要管理層關注的問題清單 |
| 6 | 需要決策的事項 | 具體的決策請求，附上選項與建議 |
| 7 | 下週計畫 | 各專案下週的主要目標 |

### 實作一：資料整合模組

**步驟 1：設計通用的資料讀取介面**

```
請幫我建立一個跨來源的工作資料整合模組：

支援以下資料來源：
1. CSV 檔案（Jira/Trello 匯出格式）：
   - 自動偵測欄位名稱（不同系統欄位名稱不同）
   - 使用 LLM 進行欄位名稱的語意映射
     （例如 "Status"、"狀態"、"state" 都映射到統一的 "task_status"）

2. 工作紀錄文字檔（.txt 或 .docx）：
   - 支援多種結構（日記格式、條列式、會議記錄格式）
   - 使用 LLM 提取結構化的任務資訊

3. Excel 時程表：
   - 讀取計畫日期 vs 實際日期

輸出統一的資料格式：
{

  "任務名稱": "",
  "所屬專案": "",

  "計畫開始日": "YYYY-MM-DD",
  "計畫結束日": "YYYY-MM-DD",
  "實際開始日": "YYYY-MM-DD",
  "實際結束日": "YYYY-MM-DD",
  "優先級": "高/中/低",
  "備註": ""
}
```

### 實作二：AI 里程碑提取與分析

**步驟 1：建立里程碑提取引擎**

在 Copilot Chat 中輸入：

```
請建立一個 sprint_analyzer.py 程式，功能如下：

1. 輸入：整合後的任務清單（上一個步驟的輸出）+ 本週工作紀錄文字

2. 計算 Sprint KPI：
   - Sprint 完成率 = 狀態為"已完成"的任務數 / 總任務數
   - 按時完成率 = 實際結束日 <= 計畫結束日 的比例
   - 阻塞率 = 狀態為"阻塞"的任務比例
   - 工時使用率（如果有工時資料）

3. 用 LLM 分析以下問題：
   Q1：本週有哪些關鍵里程碑？（里程碑名稱、意義、影響）
   Q2：有哪些風險或阻塞需要管理層關注？
   Q3：哪些任務有延遲，延遲原因是什麼？
   Q4：下週最重要的 3 件事是什麼？
   Q5：有沒有需要管理層決策的事項？

4. 輸出一個結構化的分析結果字典，供後續 PPT 生成使用
```

### 實作三：週報 PPT 自動生成

**步驟 1：設計 PPT 生成邏輯**

```
請建立一個 report_generator.py，使用 python-pptx 生成週報 PPT：

1. 載入 PPT 模板（事先設計好的 .pptx 檔，有正確的字型、配色和 Logo）

2. 生成以下頁面：

   封面頁：
   - 標題："{公司名稱} 專案週報"
   - 副標題："第 {週次} 週 | {日期範圍}"

   整體健康度頁：
   - 大圓形指示燈：綠（良好）/ 黃（需注意）/ 紅（有風險）
   - 顏色判斷規則：
     * 綠色：Sprint 完成率 >= 80% 且 阻塞率 <= 10%
     * 黃色：Sprint 完成率 60-79% 或 阻塞率 11-20%
     * 紅色：Sprint 完成率 < 60% 或 阻塞率 > 20%
   - 一句話摘要（由 LLM 生成）

   KPI 儀表板頁：
   - 對每個專案，顯示一個條狀圖（Sprint 完成率）
   - 完成率低於 70% 的標記為紅色

   里程碑頁：
   - 表格：本週達成的里程碑（里程碑名稱、達成日、重要性）
   - 表格：下週預計達成的里程碑

   風險與決策頁：
   - 風險清單（條列式）
   - 需決策事項（條列式，每項包含：問題描述、建議選項、截止日期）

3. 儲存 PPT 為："{公司名稱}_週報_W{週次}_{日期}.pptx"
```

### 實作四：自動化排程

**步驟 1：設定每週自動執行**

```
請建立 scheduler.py，設定每週五下午 4 點自動執行完整的週報生成流程：

1. 從指定資料夾讀取本週的所有資料來源檔案
2. 執行整合 → 分析 → 生成 PPT
3. 將生成的 PPT 儲存到「週報歸檔」資料夾
4. 發送郵件通知相關人員（使用第 5 堂的 Outlook 自動化）
5. 在 console 顯示執行摘要（哪些步驟成功/失敗）

使用 Python 的 schedule 套件設定排程：
    import schedule
    schedule.every().friday.at("16:00").do(執行週報流程)
```

## 程式碼範例（虛擬碼）

### 里程碑提取

```
函式 提取里程碑(任務清單, 工作紀錄文字):
    提示詞 = f"""
    你是一位資深 PM，請根據以下資訊提取本周關鍵里程碑。

    任務狀態變更（本週從進行中變為已完成的任務）：
    {過濾出本週完成的任務(任務清單)}

    工作紀錄文字：
    {工作紀錄文字}

    請輸出 JSON 格式，包含：
    [
      {{
        "名稱": "里程碑簡短標題",
        "達成日期": "YYYY-MM-DD",
        "描述": "一句話說明這個里程碑的意義",
        "影響": "對整體專案進度的影響",
        "後續行動": "這個里程碑解鎖了什麼下一步"
      }}
    ]
    """

    LLM回應 = 呼叫LLM(提示詞, 格式=JSON模式)
    里程碑清單 = 解析JSON(LLM回應)

    回傳 里程碑清單

函式 生成管理層摘要(KPI數據, 里程碑清單, 風險清單):
    提示詞 = f"""
    你是一位高階 PM，需要向 CXO 報告本週專案狀況。
    
    KPI 數據：
    {KPI數據}
    
    關鍵里程碑：
    {里程碑清單}
    
    風險事項：
    {風險清單}
    
    請生成一段 3-5 句的執行摘要，使用管理層的語言：
    - 不要用技術術語
    - 強調業務影響而非技術細節
    - 清楚指出需要關注的重點
    """

    摘要 = 呼叫LLM(提示詞)
    回傳 摘要
```

### PPT 生成

```
載入 python_pptx 套件

函式 生成週報PPT(分析結果, 模板路徑, 輸出路徑):
    簡報 = 開啟簡報(模板路徑)

    # 封面頁（模板第 0 頁）
    封面頁 = 簡報.slides[0]
    設定文字框(封面頁, "標題佔位符", f"專案週報 W{分析結果['週次']}")
    設定文字框(封面頁, "日期佔位符", 分析結果["日期範圍"])

    # 整體健康度頁（模板第 1 頁）
    健康度頁 = 簡報.slides[1]
    顏色 = 判斷健康度顏色(分析結果["KPI"])
    設定圓形顏色(健康度頁, 顏色)
    設定文字框(健康度頁, "摘要佔位符", 分析結果["管理層摘要"])

    # KPI 儀表板頁（模板第 2 頁）
    KPI頁 = 簡報.slides[2]
    對 每個專案 在 分析結果["各專案KPI"] 中:
        顏色 = "紅色" 如果 專案["完成率"] < 70 否則 "綠色"
        新增條狀圖(KPI頁, 專案名稱=專案["名稱"], 完成率=專案["完成率"], 顏色=顏色)

    # 里程碑頁（模板第 3 頁）
    里程碑頁 = 簡報.slides[3]
    里程碑表格資料 = [
        ["里程碑", "達成日期", "重要性"],
        *[[m["名稱"], m["達成日期"], m["描述"]] for m in 分析結果["里程碑"]]
    ]
    新增表格(里程碑頁, 里程碑表格資料)

    # 風險與決策頁（模板第 4 頁）
    風險頁 = 簡報.slides[4]
    對 i, 風險 在 enumerate(分析結果["風險清單"]):
        新增條列(風險頁, i, f"⚠ {風險}")
    對 i, 決策 在 enumerate(分析結果["決策事項"]):
        新增條列(風險頁, i + 10, f"🎯 {決策['問題']}\n   建議：{決策['建議']}")

    簡報.save(輸出路徑)
    顯示("✅ 週報已生成：" + 輸出路徑)
    回傳 輸出路徑

函式 判斷健康度顏色(KPI):
    完成率 = KPI["sprint完成率"]
    阻塞率 = KPI["阻塞率"]

    如果 完成率 >= 80 且 阻塞率 <= 10:
        回傳 "綠色"
    否則如果 完成率 >= 60 且 阻塞率 <= 20:
        回傳 "黃色"
    否則:
        回傳 "紅色"
```
