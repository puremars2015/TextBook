# 第05堂｜智慧通訊協作：電子郵件與 Office 流程自動化

## 學習目標

完成本堂課後，學員將能夠：

- 建立基於 Outlook 的智慧郵件自動化流程
- 使用 AI 對郵件進行語意分析，識別意圖與優先順序
- 自動化下載與解析郵件附件
- 根據郵件內容觸發對應的回覆邏輯與任務分派
- 整合郵件與行事曆，實現會議排程自動化

---

## 核心概念說明

### 1. 為什麼郵件自動化是企業的重要場景？

電子郵件是企業最主要的通訊工具之一，但也是效率瓶頸的來源：

- 平均每位知識工作者每天花費 **2.5 小時**處理電子郵件
- 大量郵件是重複性的：確認收到、請款、詢問狀態
- 重要郵件容易淹沒在垃圾訊息中，造成遺漏

AI Agent 能夠 24 小時不間斷地監控、分類、回覆並分派郵件任務。

---

### 2. Outlook 自動化的技術選項

在 Windows 環境下，操控 Outlook 主要有以下幾種方式：

| 技術 | 說明 | 適合情境 |
|------|------|---------|
| **win32com（pywin32）** | 透過 Windows COM 介面控制 Outlook 應用程式 | 本地 Outlook 桌面版 |
| **Microsoft Graph API** | 透過 Microsoft 官方 API 存取郵件、行事曆 | 雲端 M365 環境 |
| **exchangelib** | Python 套件，連接 Exchange Server | 企業 Exchange 環境 |
| **IMAP / SMTP** | 標準郵件協定，跨平台 | 非 Outlook 的郵件服務 |

本堂課主要以 **win32com** 示範本地 Outlook 操作，以及介紹 **Microsoft Graph API** 的概念作為現代化替代方案。

### 3. 郵件的語意分析

**語意分析（Semantic Analysis）**是指理解文字背後的**真實意圖**，而不只是字面上的詞彙。

**傳統規則方式 vs. AI 語意分析：**

| | 傳統關鍵字規則 | AI 語意分析 |
|-|--------------|-----------|
| 識別「催款」郵件 | 只能識別包含「請款」「付款」等關鍵字 | 能理解「我們上個月的發票還沒收到款項」也是催款意圖 |
| 識別「投訴」郵件 | 只能識別「抱怨」「不滿」等詞 | 能識別語氣委婉但明確表達不滿的郵件 |
| 多語言支援 | 需要為每種語言維護不同的規則 | 同樣的 Prompt 即可處理中、英、日文郵件 |

**郵件語意分析的典型分類任務：**

| 分析維度 | 可能的分類結果 |
|---------|--------------|
| 意圖分類 | 詢問報價、確認訂單、申請退款、投訴、一般諮詢 |
| 優先級判斷 | 緊急、一般、低優先 |
| 情感分析 | 正面、中性、負面 |
| 需要回覆？ | 是、否 |
| 對應部門 | 業務、財務、客服、技術支援 |

### 4. 附件處理自動化

郵件附件是企業資訊交換的重要媒介，常見的附件處理需求：

- **PDF**：合約、發票、報告 → 提取文字內容
- **Excel**：資料報表 → 讀取並彙整資料
- **Word**：文件草稿 → 提取內容進行摘要
- **圖片**：掃描文件 → OCR 文字識別

**附件自動化處理流程：**

```
收到郵件
    │
    ├── 是否有附件？
    │       │ 是
    │       ▼
    │   判斷附件類型
    │       ├── PDF → 使用 PDF 解析工具提取文字
    │       ├── Excel → 使用 pandas 讀取資料
    │       └── 圖片 → 使用 OCR 或 AI Vision 識別
    │       │
    │       ▼
    │   將提取內容傳給 LLM 進行分析
    │       │
    │       ▼
    │   根據分析結果觸發對應動作
    │
    └── 無附件 → 直接進行郵件內文分析
```

### 5. 行事曆同步自動化

**常見的會議排程自動化場景：**

- 客戶寄來「請問下週可以開會嗎？」→ AI 查看行事曆空檔，自動回覆可用時間
- 收到會議邀請郵件 → 自動建立行事曆事件並設定提醒
- 定期報告完成 → 自動安排下次討論會議

**行事曆操作的基本元素：**

| 元素 | 說明 |
|------|------|
| 事件標題 | 會議名稱 |
| 開始/結束時間 | 日期與時段 |
| 地點 | 實體地點或線上連結 |
| 與會者 | 邀請對象的 Email |
| 提醒 | 會議前多少分鐘提醒 |
| 重複設定 | 每週固定時段的例行會議 |

## 實作步驟（逐步教學）

### 實作一：讀取 Outlook 郵件並進行分類

**步驟 1：安裝必要套件**

```
安裝 pywin32（Outlook COM 介面）:
    pip install pywin32

安裝 openai（LLM API）:
    pip install openai
```

**步驟 2：向 GitHub Copilot 描述需求**

```
請幫我用 Python 和 pywin32 寫一個程式：
1. 連接到本地的 Microsoft Outlook
2. 讀取「收件匣」中最新的 20 封未讀郵件
3. 對每封郵件提取：寄件者、主旨、收到時間、正文內容
4. 呼叫 OpenAI API，讓 AI 判斷每封郵件的「類型」（詢問/投訴/確認/其他）
   和「優先級」（高/中/低）
5. 根據分類結果，印出分類統計報告
```

**步驟 3：理解郵件的 COM 物件結構**

Outlook 的郵件物件包含許多屬性，重點關注：
- `Subject`：郵件主旨
- `SenderEmailAddress`：寄件者地址
- `Body`：郵件內文（純文字）
- `ReceivedTime`：收到時間
- `Attachments`：附件集合
- `UnRead`：是否未讀

### 實作二：自動化回覆流程

**步驟 1：設計回覆邏輯**

```
請在上一個程式的基礎上，加入自動回覆功能：
1. 對於分類為「詢問報價」的郵件：
   - 讓 AI 根據郵件內容生成一封專業的初步回覆（說明需要進一步確認規格）
   - 將回覆草稿儲存到 Outlook 的「草稿箱」（不要直接發送）
2. 對於分類為「投訴」的郵件：
   - 標記為高優先級（紅色旗幟）
   - 轉寄給主管信箱（從環境變數讀取主管信箱）
   - 自動回覆客戶說明正在處理中
請注意：所有自動發送的郵件都要先存入草稿箱供人工審核，不要直接發送。
```

> **重要原則**：自動化郵件系統應設計為「輔助」而非「完全取代」人工判斷。關鍵郵件務必要有人工審核環節。

### 實作三：附件分析與任務分派

**步驟 1：加入附件自動處理**

```
請幫我加入附件自動處理功能：
1. 偵測郵件中的附件
2. 如果是 PDF 附件：使用 pdfplumber 套件提取文字，然後讓 AI 摘要內容
3. 如果是 Excel 附件：使用 pandas 讀取，讓 AI 描述資料的基本統計
4. 將附件內容摘要加入分類結果的報告中
5. 自動將附件儲存到對應分類的資料夾（例如：「投訴資料夾」、「報價單資料夾」）
```

**步驟 2：安裝 PDF 解析套件**

```
pip install pdfplumber
```

## 程式碼範例（虛擬碼）

### 讀取並分類 Outlook 郵件

```
載入 win32com.client 套件
載入 openai 套件

# 連接 Outlook
Outlook應用 = 啟動("Outlook.Application")
名稱空間 = Outlook應用.GetNamespace("MAPI")
收件匣 = 取得資料夾("收件匣")

分類結果清單 = []

# 讀取最新 20 封未讀郵件
未讀郵件清單 = [郵件 for 郵件 in 收件匣.Items if 郵件.UnRead == 真][:20]

對 每封郵件 在 未讀郵件清單 中:
    # 提取郵件資訊
    郵件資訊 = {
        "寄件者": 郵件.SenderEmailAddress,
        "主旨": 郵件.Subject,
        "時間": 郵件.ReceivedTime,
        "內文": 郵件.Body[:1000]  # 只取前 1000 字避免超過 Token 限制
    }

    # 呼叫 AI 分類
    提示詞 = f"""
    以下是一封電子郵件：
    主旨：{郵件資訊["主旨"]}
    內文：{郵件資訊["內文"]}

    請以 JSON 格式回傳：
    {{
        "類型": "詢問/投訴/確認訂單/請款/其他",
        "優先級": "高/中/低",
        "摘要": "20字以內的摘要",
        "建議動作": "建議的處理方式"
    }}
    """

    AI分類結果 = 呼叫LLM(提示詞)
    分類結果清單.加入({...郵件資訊, ...解析JSON(AI分類結果)})

# 輸出統計報告
顯示分類統計報告(分類結果清單)
```

### 自動回覆草稿生成

```
函式 生成回覆草稿(原始郵件, 分類結果):

    如果 分類結果.類型 == "詢問報價":
        回覆提示詞 = f"""
        客戶寄了以下詢價郵件：
        {原始郵件.內文}

        請幫我撰寫一封專業的初步回覆，要：
        1. 感謝客戶詢問
        2. 說明需要確認具體規格才能提供正式報價
        3. 請客戶提供：數量、交期需求、特殊規格要求
        4. 語氣要專業友善
        5. 用繁體中文撰寫
        """
        回覆內容 = 呼叫LLM(回覆提示詞)

        # 建立草稿
        新郵件 = Outlook應用.CreateItem(郵件類型)
        新郵件.To = 原始郵件.SenderEmailAddress
        新郵件.Subject = "Re: " + 原始郵件.Subject
        新郵件.Body = 回覆內容
        新郵件.儲存到草稿箱()  # 只存草稿，不發送

    否則如果 分類結果.類型 == "投訴":
        原始郵件.設定優先級(高)
        原始郵件.轉寄給(主管信箱)
        # ... 類似邏輯
```

### 附件自動下載與解析

```
函式 處理附件(郵件, 儲存資料夾):
    附件摘要清單 = []

    對 每個附件 在 郵件.Attachments 中:
        附件名稱 = 附件.FileName
        本地路徑 = 儲存資料夾 + "/" + 附件名稱

        # 儲存附件到本地
        附件.SaveAsFile(本地路徑)

        # 依附件類型處理
        如果 附件名稱.結尾是(".pdf"):
            文字內容 = 使用pdfplumber提取文字(本地路徑)
            摘要 = AI摘要(文字內容)
            附件摘要清單.加入({"檔名": 附件名稱, "摘要": 摘要})

        否則如果 附件名稱.結尾是(".xlsx", ".csv"):
            資料 = 使用pandas讀取(本地路徑)
            統計描述 = AI描述資料(資料.基本統計())
            附件摘要清單.加入({"檔名": 附件名稱, "摘要": 統計描述})

    回傳 附件摘要清單
```
