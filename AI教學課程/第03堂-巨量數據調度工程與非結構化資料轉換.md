# 第03堂｜巨量數據調度工程與非結構化資料轉換

## 學習目標

完成本堂課後，學員將能夠：

- 說明 AI Agent 處理大量資料的典型架構與流程
- 區別「結構化資料」與「非結構化資料」的差異
- 使用程式對試算表（Excel / CSV）進行讀取、篩選、排序與寫入

---

## 核心概念說明

### 1. 為什麼資料處理是 Agent 的核心能力？

在企業環境中，大量的工作時間花費在「整理資料」上：

- 把多份 Excel 報表合併成一份
- 從上百筆記錄中找出符合條件的資料
- 將不同格式的資料統一成標準格式

這些工作**重複性高、規則性強**，是 AI Agent 最能發揮價值的場景。

**一個典型的企業痛點：**

> 每個月月底，業務助理要花一整天，手動把 20 個業務員各自回傳的 Excel 報價單，複製貼上到一份匯總表裡，然後再計算每個業務的總業績。

如果用 Agent 處理，整個流程可以在幾分鐘內完成。

---

### 2. 結構化資料 vs. 非結構化資料

| | 結構化資料 | 非結構化資料 |
|-|-----------|-------------|
| **定義** | 有固定格式，可直接用程式讀取 | 沒有固定格式，需要解析才能使用 |
| **範例** | Excel 表格、CSV 檔、資料庫 | PDF 報告、電子郵件、掃描圖片 |
| **處理難度** | 相對容易 | 需借助 AI 或 OCR 技術 |
| **典型工作** | 篩選、排序、計算 | 資訊提取、摘要、分類 |

**從非結構化轉為結構化的典型流程：**

```
原始資料（PDF 合約、郵件內文、掃描文件）
    │
    ▼ AI / OCR 解析
提取關鍵欄位（廠商名稱、金額、日期）
    │
    ▼ 資料清理與驗證
結構化資料表（可存入 Excel 或資料庫）
    │
    ▼ 後續分析與報表生成
最終報告
```

---

### 3. Excel / CSV 自動化處理

在 Python 中，處理試算表最常用的工具是：

| 工具 | 適用情境 | 特點 |
|------|---------|------|
| **openpyxl** | 讀寫 `.xlsx` 格式的 Excel | 可保留原有格式、樣式 |
| **pandas** | 大量資料的分析與計算 | 功能強大，適合數據分析 |
| **csv 模組** | 讀寫 `.csv` 純文字表格 | 輕量，適合簡單場景 |

**試算表操作的基本動作：**

| 動作 | 說明 |
|------|------|
| 讀取（Read） | 從檔案載入資料到程式 |
| 篩選（Filter） | 只取出符合條件的列 |
| 排序（Sort） | 依照某欄位大小排列順序 |
| 計算（Calculate） | 加總、平均、計數等 |
| 寫入（Write） | 把處理結果存回檔案 |

### 4. 跨活頁簿資料整合（批次處理）

**批次處理（Batch Processing）**是指一次性處理大量同類型任務，而不是一筆一筆手動處理。

**跨活頁簿整合的典型場景：**

```
資料夾結構：
  /業務報表/
    ├── 北區業務_王大明.xlsx
    ├── 中區業務_李小華.xlsx
    ├── 南區業務_張美玲.xlsx
    └── ... (共 20 個檔案)

目標：
  自動讀取所有檔案 → 彙整到一份 匯總表.xlsx → 計算各業務業績合計
```

**批次處理的核心邏輯：**

```
步驟1：找到所有需要處理的檔案
列出資料夾內所有 .xlsx 檔案 → 得到一個檔案清單

步驟2：逐一處理每份檔案（迴圈）
對 每個檔案 在 檔案清單 中:
    讀取該檔案的資料
    提取需要的欄位
    將資料加入「總彙整清單」

步驟3：整理與計算
對 總彙整清單 進行排序、計算、格式化

步驟4：輸出結果
將整理後的資料寫入新的 Excel 檔案
```

---

### 5. AI 在資料處理中的角色

傳統的資料處理需要明確的規則（「如果欄位 A 等於 X，就…」）。但現實世界的資料往往**格式不統一**，這時 AI 就能發揮作用：

**AI 的三個關鍵角色：**

| 角色 | 說明 | 範例 |
|------|------|------|
| **格式識別** | 理解不同格式的資料代表什麼 | 「NT$12,000」「12000元」都代表 12000 元 |
| **缺失值補充** | 根據上下文推斷缺失的資訊 | 從「台北辦公室」推斷地區為「北區」 |
| **異常偵測** | 找出資料中不合理的值 | 業績填寫為負數 → 可能是錯誤 |

## 實作步驟（逐步教學）

### 實作一：讀取並篩選 Excel 資料

**步驟 1：安裝必要套件**

```
安裝 openpyxl 套件:
    pip install openpyxl

安裝 pandas 套件:
    pip install pandas openpyxl
```

**步驟 2：準備測試資料**

建立一個 `sample_data.xlsx`，包含以下欄位：

| 業務員 | 地區 | 銷售金額 | 日期 |
|--------|------|---------|------|
| 王大明 | 北區 | 50000 | 2025/07 |
| 李小華 | 中區 | 32000 | 2025/07 |
| 張美玲 | 南區 | 68000 | 2025/07 |
| ... | ... | ... | ... |

**步驟 3：向 GitHub Copilot 描述需求**

```
請幫我用 Python 和 pandas 寫一個程式：
1. 讀取 sample_data.xlsx 的資料
2. 篩選出「銷售金額」超過 40000 的記錄
3. 依照「銷售金額」由高到低排序
4. 計算所有業務員的銷售金額總計
5. 把結果輸出到新的 filtered_result.xlsx 檔案
```

### 實作二：批次彙整多份活頁簿

**步驟 1：準備多份測試 Excel 檔案**

在 `報表資料/` 資料夾中建立多個 xlsx 檔案（可手動建立 3-5 個模擬檔案）。

**步驟 2：向 GitHub Copilot 描述需求**

```
請幫我用 Python 寫一個批次處理程式：
1. 掃描「報表資料」資料夾內所有的 .xlsx 檔案
2. 讀取每個檔案的資料（假設欄位格式相同）
3. 把所有資料合併到一個 DataFrame
4. 新增一個「來源檔案」欄位，記錄每筆資料來自哪個檔案
5. 依照業務員計算各自的銷售金額加總
6. 將匯總結果輸出到「月份彙整報表.xlsx」
```

**步驟 3：加入 AI 輔助的格式識別**

在程式中加入 LLM 呼叫，讓 AI 幫助你處理格式不一致的問題：
- 有些欄位名稱不同（「業務」vs「業務員」）
- 有些金額格式不同（「$50,000」vs「50000」）

### 實作三：AI Agent 驅動的資料清理

**步驟 1：描述整體 Agent 流程**

```
我要建立一個資料清理 Agent：
1. 讀取一份包含格式不一致資料的 Excel 檔
2. 對每一列資料，呼叫 LLM API 進行格式標準化
3. 將清理後的資料寫回新的 Excel 檔案
4. 如果某一列資料 AI 無法確定，標記為「需人工審查」
請幫我設計這個程式的架構
```

## 程式碼範例（虛擬碼）

### 讀取與篩選試算表

```
載入 pandas 套件

# 讀取 Excel 檔案
資料表 = 讀取Excel("sample_data.xlsx", 工作表="Sheet1")

# 查看資料基本資訊
顯示 資料表的前5行
顯示 資料表的欄位名稱
顯示 資料表的資料筆數

# 篩選：銷售金額超過 40000
篩選結果 = 資料表[ 資料表["銷售金額"] > 40000 ]

# 排序：由高到低
排序結果 = 篩選結果.依照("銷售金額", 由大到小=真)

# 計算加總
總計 = 篩選結果["銷售金額"].加總()
顯示 f"篩選後總計：{總計}"

# 輸出到新 Excel 檔案
排序結果.寫入Excel("filtered_result.xlsx", 不含索引=真)
顯示 "結果已儲存"
```

### 批次彙整多份活頁簿

```
載入 pandas, os, pathlib 套件

資料夾路徑 = "報表資料/"
所有資料 = []  # 用來收集所有檔案的資料

# 掃描資料夾內所有 xlsx 檔案
對 每個檔案 在 列出資料夾(資料夾路徑, 篩選="*.xlsx") 中:
    顯示 f"正在處理：{檔案名稱}..."

    # 讀取當前檔案
    當前資料 = 讀取Excel(檔案路徑)

    # 加入「來源檔案」欄位
    當前資料["來源檔案"] = 檔案名稱

    # 加入總清單
    所有資料.加入(當前資料)

# 合併所有資料
彙整資料 = 合併所有(所有資料)

# 依業務員計算加總
業績彙總 = 彙整資料.依照("業務員").計算加總("銷售金額")

# 輸出
業績彙總.寫入Excel("月份彙整報表.xlsx")
顯示 f"完成！共處理 {len(所有資料)} 個檔案"
```

### AI 輔助的格式標準化

```
函式 標準化欄位(原始值, 欄位類型):
    提示詞 = f"""
    我有一筆資料："{原始值}"
    這是「{欄位類型}」欄位的值。
    請將它轉換為標準格式：
    - 如果是金額：只回傳數字（不含符號與逗號）
    - 如果是日期：轉換為 YYYY-MM-DD 格式
    - 如果無法判斷：只回傳「UNKNOWN」
    只回傳結果，不要說明。
    """
    回傳 呼叫LLM(提示詞)

# 對每一列資料進行標準化
對 每一列資料 在 資料表 中:
    資料["銷售金額"] = 標準化欄位(資料["銷售金額"], "金額")
    資料["日期"]   = 標準化欄位(資料["日期"], "日期")

    如果 任何欄位 == "UNKNOWN":
        資料["備註"] = "需人工審查"
```
