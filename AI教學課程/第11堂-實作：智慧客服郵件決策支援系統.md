# 第11堂｜實作：智慧客服郵件決策支援系統

## 學習目標

完成本堂課後，學員將能夠：

- 建立基於 AI 語意理解的郵件自動分類與優先排序系統
- 模擬真實客服場景，自動生成符合情境的回覆草稿
- 設計複雜異議處理的決策樹，讓 Agent 能導航至正確的回覆策略
- 建立關鍵資訊自動標記機制，加速人工審核效率

---

## 核心概念說明

### 1. 客服自動化的商業價值

**數字說話：**
- 企業客服部門平均每天處理數百到數千封郵件
- 重複性問題（FAQ）通常佔來信量的 60-70%
- 人工閱讀並回覆每封郵件平均需要 5-15 分鐘

**AI 可以解決的痛點：**

| 痛點 | AI 解決方案 |
|------|------------|
| 來信量大、人力不足 | 自動分類 + 草稿生成，大幅縮短每封的處理時間 |
| 緊急信件淹沒在一般信件中 | AI 自動識別優先級，確保緊急案件不被遺漏 |
| 回覆品質因人而異 | AI 提供標準化草稿，確保品質一致 |
| 夜間或假日無法即時回覆 | 24小時自動發送初步響應，告知客戶已收到並將跟進 |

---

### 2. 郵件分類的維度設計

一個有效的客服郵件分類系統需要多個維度同時分析：

**維度 1：意圖分類（Intent Classification）**

| 意圖類型 | 說明 | 建議處理時限 |
|---------|------|------------|
| 產品詢問 | 詢問功能、規格、價格 | 24小時 |
| 訂單狀態 | 查詢訂單進度 | 4小時 |
| 退換貨申請 | 要求退款或換貨 | 2小時 |
| 投訴/抱怨 | 對產品或服務不滿 | 1小時 |
| 緊急問題 | 系統故障、安全問題 | 立即 |
| 讚美/感謝 | 正面回饋 | 48小時 |
| 垃圾郵件 | 廣告或無關信件 | 忽略 |

**維度 2：情感分析（Sentiment Analysis）**

| 情感程度 | 判斷依據 | 影響策略 |
|---------|---------|---------|
| 強烈負面 | 語氣激烈、出現威脅或法律字眼 | 立即升級，主管介入 |
| 輕微負面 | 不滿但語氣平和 | 優先處理，同理心回覆 |
| 中性 | 無明顯情緒 | 標準處理 |
| 正面 | 感謝、讚美 | 標準回覆，可回覆感謝 |

**維度 3：複雜程度**

| 複雜程度 | 說明 | 處理方式 |
|---------|------|---------|
| 可自動化 | 標準問題，有固定答案 | AI 直接生成回覆草稿 |
| 需人工判斷 | 個案處理，需查詢資料系統 | AI 提供資訊，人工決策 |
| 需升級 | 超出一般客服權限 | 轉交主管或專責部門 |

### 3. 異議處理的決策樹設計

**異議處理（Complaint Handling）**是客服中最具挑戰性的場景，需要：
1. 準確理解客戶的不滿點
2. 表達同理心
3. 提供可接受的解決方案
4. 確保客戶感到被重視

**異議處理的決策流程：**

```
收到投訴郵件
    │
    ▼ AI 分析：問題類型是什麼？
    ├── 產品品質問題
    │       │
    │       ├── 在保固期內？是 → 提供免費維修/換貨選項
    │       └── 超過保固期？是 → 提供付費維修選項，視情況提供折扣
    │
    ├── 服務態度問題
    │       │
    │       └── 升級主管，致歉並調查內部流程
    │
    ├── 物流延誤問題
    │       │
    │       ├── 公司責任？是 → 致歉 + 追蹤 + 補償方案
    │       └── 物流公司責任？是 → 告知原因 + 協助追蹤
    │
    └── 未達廣告宣稱的問題
            │
            └── 法務確認後再回覆，避免直接承認
```

### 4. 關鍵資訊自動標記

AI 在閱讀郵件後，應自動提取並標記以下關鍵資訊，讓人工審核者能快速掌握重點：

```
郵件摘要卡：
┌─────────────────────────────────────────┐
│ 意圖：退換貨申請       優先級：高 (🔴)  │
│ 情感：強烈負面          複雜度：需人工   │
├─────────────────────────────────────────┤
│ 關鍵資訊：                               │
│ • 訂單號：#2025-07-1234                  │
│ • 購買日期：2025/07/10                   │
│ • 問題：產品有異味、懷疑品質問題          │
│ • 客戶聲稱已拍照存證                     │
│ • 提及「若不處理將到消費者保護機構申訴」  │
├─────────────────────────────────────────┤
│ AI 建議：升級主管，24小時內必須回覆      │
└─────────────────────────────────────────┘
```

### 5. 資料安全與個資保護

客服郵件通常包含大量個人資料，在使用 AI 處理時需特別注意：

**重要原則：**

| 原則 | 說明 |
|------|------|
| **最小化傳送** | 只傳送 AI 分析所需的資訊，刪除不必要的個資 |
| **去識別化** | 在傳送前將姓名、電話等個資替換為代碼 |
| **記錄存取** | 記錄哪些郵件被 AI 處理，保留稽核軌跡 |
| **員工教育** | 確保所有使用人員了解個資保護規範 |
| **使用條款確認** | 確認 LLM 服務商的資料使用政策是否符合公司合規要求 |

## 實作步驟（逐步教學）

### 實作一：建立郵件分類引擎

**步驟 1：設計分類 Prompt**

向 GitHub Copilot 描述：
```
請幫我建立一個郵件分類函式：
1. 輸入：郵件主旨 + 郵件內文（限制內文最多 500 字，超過截斷）
2. 呼叫 LLM 進行以下分析，以 JSON 格式回傳：
   {
     "intent": "產品詢問/訂單狀態/退換貨/投訴/緊急/讚美/垃圾郵件",
     "sentiment": "強烈負面/輕微負面/中性/正面",
     "priority": "立即/高/中/低",
     "complexity": "可自動化/需人工/需升級",
     "summary": "20字以內的摘要",
     "key_info": {
       "order_number": "訂單號（若有）",
       "product": "提到的產品（若有）",
       "main_issue": "核心問題描述",
       "escalation_signal": "有無升級警示（法律威脅/媒體威脅/個資要求等）"
     },
     "suggested_action": "建議的處理方式"
   }
3. 個資保護：在傳給 LLM 之前，將郵件中的姓名替換為「[客戶]」、電話替換為「[電話]」
```

### 實作二：建立回覆草稿生成器

**步驟 1：設計多情境的回覆 Prompt 庫**

```
請幫我建立一個回覆草稿生成函式，需要支援以下情境：
1. 標準產品詢問回覆（友善、提供產品資訊、引導到購買）
2. 訂單狀態回覆（提供追蹤資訊的格式，請人工填入實際資料）
3. 退換貨投訴回覆（同理心開頭 + 標準處理流程 + 後續追蹤承諾）
4. 緊急問題初步回覆（確認收到 + 承諾後續跟進時間）
5. 讚美感謝回覆（簡短感謝 + 歡迎再次光臨）

要求：
- 語氣要專業但不失溫度
- 不要承諾任何具體的補償金額（這需要人工決定）
- 回覆要標記「[需人工填入的部分]」讓審核者清楚知道什麼地方需要確認
- 用繁體中文，語氣符合台灣商務慣例
```

### 實作三：整合完整的客服決策支援介面

**步驟 1：建立處理流程主程式**

```
請幫我整合以上功能，建立完整的客服支援工具：
1. 監控指定 Outlook 資料夾的新郵件（或讀取測試郵件清單）
2. 對每封郵件執行分類分析
3. 依優先級排序，高優先的排在最前面
4. 生成每封郵件的「摘要卡」（包含分類結果和關鍵資訊）
5. 對可自動化的郵件，生成回覆草稿並存入草稿箱
6. 對需升級的郵件，發送內部通知給主管（使用 LINE Notify 或郵件）
7. 將所有處理結果記錄到 Excel（含：處理時間、分類結果、是否有草稿）
8. 生成每日客服情況統計報表
```

## 程式碼範例（虛擬碼）

### 郵件分類函式（含個資保護）

```
載入 re（正規表達式）套件

函式 去除個資(郵件內文):
    # 替換常見的個資格式
    結果 = 郵件內文
    結果 = 取代符合 "0[89]\d{8}" → "[電話]"   # 手機號碼
    結果 = 取代符合 "\d{4}-\d{4}" → "[電話]"   # 市話格式
    結果 = 取代符合 "身份證[字號:\s]*[A-Z]\d{9}" → "[身分證]"
    結果 = 取代符合 "[A-Za-z0-9._%]+@[A-Za-z0-9.]+\.[A-Za-z]+" → "[Email]"
    回傳 結果

函式 分析郵件(主旨, 內文):
    # 個資保護：去除識別資訊
    安全內文 = 去除個資(內文[:500])  # 限制500字

    提示詞 = f"""
    你是一個客服郵件分析助理。請分析以下郵件並以 JSON 格式回傳結果。

    郵件主旨：{主旨}
    郵件內文：{安全內文}

    分析結果格式：
    {{
        "intent": "產品詢問|訂單狀態|退換貨|投訴|緊急|讚美|垃圾郵件",
        "sentiment": "強烈負面|輕微負面|中性|正面",
        "priority": "立即|高|中|低",
        "complexity": "可自動化|需人工|需升級",
        "summary": "核心問題20字摘要",
        "key_info": {{
            "main_issue": "問題描述",
            "escalation_signal": true/false,
            "escalation_reason": "升級原因（若有）"
        }},
        "suggested_action": "建議處理方式"
    }}
    只回傳 JSON。
    """

    嘗試:
        AI回應 = 呼叫LLM(提示詞)
        回傳 解析JSON(AI回應)
    如果發生錯誤:
        記錄錯誤(主旨)
        回傳 {"priority": "中", "complexity": "需人工", "summary": "分析失敗，請人工審核"}
```

### 回覆草稿生成

```
回覆模板庫 = {
    "退換貨": """
        感謝您聯繫我們。

        我完全理解您目前的困擾，非常抱歉我們的產品/服務讓您感到不便。

        針對您反映的 [主要問題]，我們將按照以下流程為您處理：
        1. [需人工填入：退換貨具體流程步驟]
        2. 預計處理時間：[需人工確認時間]
        3. [需人工填入：補償方案（若有）]

        我們將在 [需人工確認時間] 之前給您明確的回覆。
        若您有任何疑問，請隨時回覆此郵件。

        再次為給您造成的不便致上最誠摯的歉意。
    """,
    # ... 其他模板
}

函式 生成回覆草稿(郵件分析結果, 原始郵件摘要):
    意圖 = 郵件分析結果["intent"]
    情感 = 郵件分析結果["sentiment"]
    主要問題 = 郵件分析結果["key_info"]["main_issue"]

    如果 意圖 == "垃圾郵件":
        回傳 None  # 垃圾郵件不回覆

    如果 郵件分析結果["complexity"] == "需升級":
        # 不自動生成回覆，轉人工
        通知主管(郵件分析結果)
        回傳 None

    # 使用 AI 生成個性化草稿
    提示詞 = f"""
    請根據以下資訊，生成一封客服回覆郵件草稿：

    客戶意圖：{意圖}
    客戶情感狀態：{情感}
    核心問題：{主要問題}

    參考模板：
    {回覆模板庫.get(意圖, 回覆模板庫["標準"])}

    要求：
    - 開頭需要針對客戶情感做適當的回應（負面情感需要先道歉/同理）
    - 用繁體中文，語氣專業而溫暖
    - 用 [需人工填入：說明] 標記所有需要人工確認的地方
    - 不要承諾具體金額或時間，那些都要 [需人工填入]
    """

    草稿 = 呼叫LLM(提示詞)
    回傳 草稿
```
