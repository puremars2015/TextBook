## 核心概念說明

### 1. 企業供應鏈數據管理的痛點

**典型企業場景：**

一家製造業公司，每次採購物料時需要：
1. 向 10-20 家廠商詢價
2. 各廠商用自己的格式回傳報價單（Excel、PDF、Word 都有）
3. 採購人員逐一打開報價單，手動複製關鍵資料到比較表
4. 計算各廠商的綜合評分（價格、交期、品質等）
5. 製作比較報告提交主管審核

**這個流程的問題：**

| 問題 | 影響 |
|------|------|
| 格式不統一 | 每份報價單都要重新閱讀理解結構 |
| 手動複製資料 | 容易出現輸入錯誤 |
| 比較邏輯複雜 | 要同時考慮多個維度，容易遺漏 |
| 歸檔不系統 | 找歷史報價困難，版本管理混亂 |
| 耗時 | 採購人員的大量時間花在這些低附加價值的工作 |

---

### 2. 異質資料提取的挑戰

**為什麼廠商的報價單格式不統一？**

- 各廠商有自己的 ERP 系統，匯出格式各異
- 有些廠商習慣用 Excel 模板，表格結構不同
- 有些廠商只用 PDF，甚至是掃描紙本
- 欄位名稱不一樣（「單價」vs「每件価格」vs「Unit Price」）
- 計算邏輯不同（含稅/不含稅、是否包含運費）

**AI 在異質資料提取的優勢：**

傳統的程式處理異質資料需要為每種格式寫獨立的解析邏輯（維護成本極高），而 LLM 能夠**理解各種格式的語意**，直接提取結果。

---

### 3. KPI 指標設計

比較廠商時，通常不能只看價格，需要考慮多個維度：

**採購 KPI 評估框架（範例）：**

| KPI 指標 | 說明 | 權重（可自訂） |
|---------|------|--------------|
| 單價 | 每件/每批的價格 | 40% |
| 交期 | 從下單到交貨的天數 | 25% |
| MOQ | 最低訂購量（影響彈性） | 10% |
| 付款條件 | 幾天內付款，是否有折扣 | 10% |
| 品質等級 | ISO 認證、過去的不良率 | 15% |

**KPI 加權評分計算：**

```
廠商總分 = 
    (單價分數 × 40%) +
    (交期分數 × 25%) +
    (MOQ分數 × 10%) +
    (付款條件分數 × 10%) +
    (品質等級分數 × 15%)

各指標的分數計算：
    單價分數 = 100 × (最低單價 / 該廠商單價)
    交期分數 = 100 × (最短交期 / 該廠商交期)
    ...
```

---

### 4. 報價單歸檔系統設計

一個好的報價單歸檔系統需要：

**資料夾結構設計：**

```
/供應商報價管理/
    /[廠商名稱]/
        /[詢價品項]/
            /[年份]/
                原始報價單_2025Q3_V1.pdf
                原始報價單_2025Q3_V2.xlsx  ← 廠商修改後的版本
                解析結果_2025Q3.json        ← AI 提取的結構化資料
                比較報表_2025Q3.xlsx        ← 最終的比較結果
```

**版本管理的重要性：**

廠商有時會在報價後調整價格，版本管理讓你能：
- 追蹤哪個版本的報價最終被採用
- 記錄廠商的報價歷史，用於下次議價的依據
- 稽核時提供完整的報價紀錄

---

### 5. 供應鏈場景的特殊考量

在供應鏈場景中，AI 自動化需要特別注意：

**資料準確性要求極高：**

| 風險 | 說明 | 控制措施 |
|------|------|---------|
| AI 提取錯誤 | 金額、數量提取有誤 | 所有提取結果需人工確認 |
| 匯率換算 | 外幣報價需要正確換算 | 匯率來源要可追溯 |
| 幣別混淆 | 同一份報價單可能混用幣別 | 明確標記每個金額的幣別 |
| 單位不統一 | kg vs g、pcs vs 箱 | 統一換算為標準單位 |

### 實作一：報價單資料提取引擎

**步驟 1：準備測試資料**

建立 3-5 份格式不同的模擬報價單（可以用 Excel 或 Word 手動建立），確保它們有不同的結構但包含相同類型的資訊：
- 廠商名稱與聯絡資訊
- 報價品項（品名、規格、數量、單價）
- 交期、付款條件、有效期限
- 備註事項

**步驟 2：向 GitHub Copilot 描述提取引擎**

```
請幫我建立一個報價單資料提取系統：

1. 支援的輸入格式：
   - Excel (.xlsx, .xls)：使用 pandas 讀取
   - PDF (.pdf)：使用 pdfplumber 提取文字
   - Word (.docx)：使用 python-docx 提取文字

2. 對提取的內容，呼叫 LLM 進行結構化提取，回傳 JSON：
   {
     "廠商資訊": {
       "廠商名稱": "",
       "聯絡人": "",
       "Email": "",
       "電話": ""
     },
     "報價日期": "",
     "有效期限": "",
     "幣別": "TWD/USD/EUR",
     "品項": [
       {
         "品名": "",
         "規格": "",
         "數量": 0,
         "單位": "",
         "單價": 0,
         "小計": 0
       }
     ],
     "交期": "",
     "付款條件": "",
     "是否含稅": true/false,
     "備註": "",
     "AI信心度": "高/中/低"  ← AI 對提取結果準確性的自我評估
   }

3. 若 AI 信心度為「低」，在結果中標記 "需人工確認": true
```

### 實作二：多廠商比較與 KPI 評分

**步驟 1：設計 KPI 評分函式**

```
請幫我建立一個多廠商 KPI 比較系統：

1. 輸入：多個廠商的結構化報價資料（字典清單）
2. 可自訂的 KPI 權重設定（從 config.json 讀取）
3. 計算每個廠商的各指標分數和加權總分
4. 輸出一份 Excel 比較表，包含：
   - 第一頁「廠商比較總覽」：所有廠商並排比較，最低價格和最高分數自動標綠色
   - 第二頁「品項詳細比較」：每個品項的各廠商單價比較
   - 第三頁「KPI 雷達圖」：視覺化各廠商在不同維度的表現
   - 第四頁「採購建議」：AI 根據數據給出的採購建議

5. 加入以下保護機制：
   - 所有貨幣自動換算為台幣（匯率從環境變數讀取）
   - 交期如果是「4-6週」、「30 天」等不同格式，統一換算為天數
   - 如果某廠商的某個欄位是 "UNKNOWN"，在比較表中特別標記
```

### 實作三：自動歸檔系統

**步驟 1：建立歸檔流程**

```
請幫我建立一個報價單自動歸檔系統：

1. 監控「待處理報價單」資料夾（使用 watchdog）
2. 偵測到新檔案時：
   a. 提取報價資料（呼叫實作一的函式）
   b. 自動建立歸檔資料夾：/報價管理/{廠商名稱}/{詢價品項}/{年季}/
   c. 將原始檔案複製到歸檔資料夾（自動加上版本號：_V1, _V2...）
   d. 將提取結果存為 JSON 檔案
   e. 更新「報價總覽」Excel（所有廠商所有批次的彙整表）

3. 每次新增報價時，自動：
   a. 與同批次的其他廠商報價比較
   b. 如果比最低價高超過 20%，發出 "價格偏高" 提醒
   c. 若達到 3 份以上報價，自動生成初步比較報告
```

## 程式碼範例（虛擬碼）

### 報價單資料提取

```
載入 pdfplumber, pandas, docx, openai, json 套件

函式 讀取檔案內容(檔案路徑):
    副檔名 = 取得副檔名(檔案路徑).轉小寫()

    如果 副檔名 在 [".xlsx", ".xls"]:
        資料 = pandas.讀取Excel(檔案路徑)
        回傳 資料.轉為字串()  # 將表格轉為文字
    否則如果 副檔名 == ".pdf":
        使用 pdfplumber.open(檔案路徑) 作為 pdf:
            全文 = ""
            對 每頁 在 pdf.pages 中:
                全文 += 每頁.extract_text()
            回傳 全文
    否則如果 副檔名 == ".docx":
        文件 = docx.Document(檔案路徑)
        全文 = "\n".join([段落.text for 段落 in 文件.paragraphs])
        回傳 全文
    否則:
        拋出錯誤(f"不支援的檔案格式：{副檔名}")

函式 提取報價資料(檔案路徑):
    # 讀取檔案
    原始內容 = 讀取檔案內容(檔案路徑)

    # AI 結構化提取
    提示詞 = f"""
    以下是一份廠商報價單的內容：
    {原始內容}

    請提取所有的報價資訊，以 JSON 格式回傳。
    格式如下：[JSON Schema 定義]
    
    特別注意：
    - 若資訊不明確，填入 "UNKNOWN" 並將 AI信心度 設為「低」
    - 金額填入數字（不含貨幣符號）
    - 日期統一格式為 YYYY-MM-DD
    只回傳 JSON，不要說明。
    """

    回應 = 呼叫LLM(提示詞, 格式=JSON模式)
    結果 = 解析JSON(回應)
    結果["來源檔案"] = 檔案路徑
    結果["提取時間"] = 現在時間()

    # 標記需人工確認的欄位
    如果 結果["AI信心度"] == "低":
        結果["需人工確認"] = 真
    
    回傳 結果
```

### KPI 評分計算

```
函式 計算KPI評分(廠商報價清單, KPI權重設定):
    評分結果 = []

    # 計算各指標的最佳值（用來算相對分數）
    最低單價 = 找最小值([廠商["品項"][0]["單價"] for 廠商 in 廠商報價清單])
    最短交期 = 找最小值([轉換為天數(廠商["交期"]) for 廠商 in 廠商報價清單])

    對 每個廠商 在 廠商報價清單 中:
        各項分數 = {}

        # 單價分數（越低越好）
        廠商單價 = 廠商["品項"][0]["單價"]
        各項分數["單價"] = (最低單價 / 廠商單價) * 100

        # 交期分數（越短越好）
        廠商交期 = 轉換為天數(廠商["交期"])
        各項分數["交期"] = (最短交期 / 廠商交期) * 100

        # 加權計算總分
        總分 = 0
        對 每個KPI, 分數 在 各項分數.items():
            權重 = KPI權重設定.get(KPI, 0)
            總分 += 分數 * 權重 / 100

        評分結果.加入({
            "廠商名稱": 廠商["廠商資訊"]["廠商名稱"],
            "各項分數": 各項分數,
            "加權總分": round(總分, 2),
            "推薦": 總分 >= 80  # 80分以上標記為推薦
        })

    # 依總分排序
    回傳 排序(評分結果, 依照="加權總分", 降序=真)
```
