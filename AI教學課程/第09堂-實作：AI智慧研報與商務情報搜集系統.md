# 第09堂｜實作：AI 智慧研報與商務情報搜集系統

## 學習目標

完成本堂課後，學員將能夠：

- 建立自動化的資料檢索與分析工作流
- 整合多個資料來源（新聞、財經網站、公開資料庫）進行情報搜集
- 使用 AI 將大量非結構化情報彙整為有商業洞察的研報
- 輸出格式化的分析文案與報告文件

---

## 核心概念說明

### 1. 什麼是商務情報（Business Intelligence）？

**商務情報（BI）**是指透過有系統地蒐集、分析和解讀資訊，幫助企業做出更好的決策。

**情報搜集的典型應用場景：**

| 場景 | 需要搜集的資訊 | 使用目的 |
|------|--------------|---------|
| 競品分析 | 競爭對手的定價、新聞、人才動向 | 調整自身策略 |
| 市場研究 | 產業趨勢、消費者輿論、法規動向 | 投資或進入新市場的決策 |
| 供應商評估 | 廠商公司資訊、財務狀況、客戶評價 | 選擇可靠的合作夥伴 |
| 客戶洞察 | 客戶公司近期動態、新聞、財報 | 銷售前準備、個人化服務 |

**傳統情報搜集 vs. AI 自動化：**

| | 傳統方式 | AI 自動化 |
|-|--------|---------|
| 耗時 | 數小時到數天 | 數分鐘 |
| 覆蓋範圍 | 有限（人力限制） | 可掃描大量來源 |
| 品質一致性 | 因人而異 | 格式一致 |
| 即時性 | 有延遲 | 可近乎即時 |

---

### 2. 自動化資料檢索的技術架構

**情報搜集 Agent 的資料流：**

```
任務輸入（搜集主題、關鍵字、時間範圍）
    │
    ▼ 多來源資料擷取
    ├── 新聞 API（Google News、NewsAPI）
    ├── Web 搜尋（Bing Search API、SerpAPI）
    ├── 社群媒體（Twitter/X API）
    ├── 公開資料庫（政府資料、財政部）
    └── 目標網站（Playwright 爬取）
    │
    ▼ 資料去重與篩選
過濾掉重複或不相關的資訊
    │
    ▼ AI 分析與彙整
    ├── 摘要每篇文章的重點
    ├── 識別跨文章的共同主題
    ├── 找出潛在的商業含義
    └── 排列資訊的重要性
    │
    ▼ 報告生成
結構化的研報文件（Word / PDF / HTML）
```

---

### 3. NewsAPI 與搜尋 API

**NewsAPI** 是一個常用的新聞資料 API，提供來自全球數千個媒體的最新新聞。

**NewsAPI 的基本用法：**

```
請求參數：
    關鍵字 (q): 要搜尋的主題
    語言 (language): zh（中文）、en（英文）
    從此日期 (from): 搜尋的開始日期
    排序方式 (sortBy): publishedAt（最新）、relevancy（相關度）

回應格式：
    [
        {
            "title": "文章標題",
            "source": {"name": "媒體名稱"},
            "publishedAt": "發布時間",
            "description": "文章摘要",
            "url": "原文連結"
        },
        ...
    ]
```

**SerpAPI（搜尋引擎結果 API）：**

SerpAPI 可以以程式方式取得 Google 搜尋結果，適合需要更廣泛資料來源的情境。

### 4. 研報的結構設計

一份有商業價值的研報通常包含以下結構：

```
[研報標題]
[搜集時間 / 分析範圍]

一、執行摘要（Executive Summary）
    - 3-5 條最重要的發現
    - 主要結論與建議

二、主題概覽
    - 背景說明
    - 本次分析範圍

三、資料來源分析
    - 共蒐集幾篇文章 / 幾個來源
    - 主要媒體分佈
    - 時間分佈

四、核心發現（依重要性排列）
    - 發現 1：[標題] + 分析 + 相關來源連結
    - 發現 2：...
    - 發現 3：...

五、趨勢分析
    - 近期趨勢走向
    - 潛在的機會與威脅

六、建議行動
    - 短期（1-3 個月）可採取的行動
    - 中長期策略方向

七、附錄：原始資料來源清單
```

---

### 5. AI 分析的 Prompt 設計

研報的核心價值在於「洞察（Insight）」，而不只是「摘要（Summary）」。

**摘要 vs. 洞察的差別：**

| | 摘要 | 洞察 |
|-|------|------|
| 說明什麼 | 「這篇文章說了 A, B, C」 | 「綜合三篇報導，可以發現 XX 公司正在做 Y，這意味著市場將往 Z 方向演變」 |
| 價值 | 省去閱讀原文的時間 | 提供決策依據 |

**設計「洞察型」Prompt 的技巧：**

```
不好的 Prompt：
「請摘要以下新聞文章。」

好的 Prompt：
「以下是過去一週關於『電動車電池市場』的 10 篇新聞。
請以一位產業分析師的角度，分析：
1. 這些新聞反映出什麼重要的市場趨勢？
2. 有哪些企業的動作值得特別關注？為什麼？
3. 這對正在考慮進入這個市場的公司意味著什麼機會或風險？
請提供具體的、可行動的建議，不要只是描述事實。」
```

## 實作步驟（逐步教學）

### 實作一：建立新聞搜集模組

**步驟 1：申請 NewsAPI Key**
1. 前往 [https://newsapi.org/](https://newsapi.org/) 申請免費帳號
2. 取得 API Key，設定為環境變數 `NEWS_API_KEY`

**步驟 2：向 GitHub Copilot 描述需求**

```
請幫我用 Python 建立一個新聞搜集模組：
1. 函式名稱：fetch_news(keywords, days=7, language="zh", max_results=20)
2. 使用 NewsAPI 搜尋指定關鍵字在最近 N 天的新聞
3. 對每篇新聞回傳：標題、媒體名稱、發布時間、內文摘要、連結
4. 過濾掉：重複的標題、內容為 None 的文章
5. 儲存結果到一個 JSON 檔案（以關鍵字和日期命名）
6. API Key 從環境變數讀取
```

**步驟 3：測試搜集結果**

搜尋一個你工作相關的產業主題，檢視：
- 取得了多少篇文章？
- 有沒有重複或不相關的結果？
- 資料的時間分佈如何？

---

### 實作二：建立 AI 分析與洞察引擎

**步驟 1：設計洞察分析 Prompt**

```
請幫我建立一個分析函式：
1. 輸入：一個包含多篇新聞摘要的清單（每篇有標題、摘要、時間）
2. 分析主題：從函式參數傳入（例如：「台灣半導體產業」）
3. 讓 AI 以產業分析師角度輸出：
   a. 執行摘要（3-5個重點，條列式）
   b. 三個最重要的市場觀察（附說明）
   c. 潛在的機會與風險各 2 條
   d. 建議企業短期可採行的 2 個行動
4. 如果文章太多超過 Token 限制，分批處理後再彙整
5. 回傳結構化的字典格式
```

---

### 實作三：生成完整研報文件

**步驟 1：安裝 Word 文件生成套件**

```
pip install python-docx
```

**步驟 2：向 Copilot 描述報告生成需求**

```
請幫我建立一個研報文件生成函式：
1. 輸入：搜集到的新聞資料 + AI 分析結果
2. 輸出：一份格式完整的 Word 文件（.docx）
3. 文件結構：
   - 封面：主題名稱、分析日期、資料來源筆數
   - 執行摘要
   - 核心發現（每項有標題、AI分析、原始來源連結）
   - 趨勢分析與建議
   - 附錄：原始資料清單
4. 格式要求：
   - 標題使用 Heading 1/2/3 樣式
   - 重點文字加粗
   - 頁首顯示標題，頁尾顯示頁碼
```

## 程式碼範例（虛擬碼）

### 新聞搜集模組

```
載入 requests, json, datetime, os 套件

函式 搜集新聞(關鍵字清單, 天數=7, 語言="zh", 最多篇數=20):
    API_KEY = 從環境變數讀取("NEWS_API_KEY")
    結束日期 = 今天()
    開始日期 = 今天() - 天數天

    所有文章 = []
    已見過的標題 = 集合()  # 用來去重

    對 每個關鍵字 在 關鍵字清單 中:
        嘗試:
            回應 = 呼叫NewsAPI(
                q=關鍵字,
                from=開始日期,
                to=結束日期,
                language=語言,
                sortBy="relevancy",
                pageSize=最多篇數
            )

            對 每篇文章 在 回應["articles"] 中:
                如果 文章["description"] 為 None: 跳過
                如果 文章["title"] 在 已見過的標題 中: 跳過  # 去重

                已見過的標題.加入(文章["title"])
                所有文章.加入({
                    "標題": 文章["title"],
                    "媒體": 文章["source"]["name"],
                    "時間": 文章["publishedAt"],
                    "摘要": 文章["description"],
                    "連結": 文章["url"]
                })

        如果發生錯誤:
            記錄錯誤(關鍵字)

    # 儲存到 JSON
    檔案名 = f"新聞搜集_{關鍵字清單[0]}_{今天日期}.json"
    寫入JSON(所有文章, 檔案名)
    顯示 f"共搜集 {len(所有文章)} 篇文章"
    回傳 所有文章
```

### AI 洞察分析函式

```
函式 分析產業情報(文章清單, 分析主題):
    # 組合所有文章摘要（注意 Token 長度限制）
    文章彙整 = ""
    對 i, 文章 在 enumerate(文章清單):
        文章彙整 += f"""
        [{i+1}] {文章["標題"]}（{文章["媒體"]}，{文章["時間"]}）
        {文章["摘要"]}
        """

    分析提示詞 = f"""
    你是一位資深產業分析師，請分析以下關於「{分析主題}」的新聞，
    提供具有商業價值的洞察。

    === 新聞資料 ===
    {文章彙整}
    === 新聞資料結束 ===

    請以結構化方式輸出以下分析（使用繁體中文）：

    1. 執行摘要（3-5個最重要發現，條列式）
    2. 三大市場觀察（每項：觀察標題 + 2-3句分析說明）
    3. 機會分析（2條，說明對企業的潛在機會）
    4. 風險提示（2條，說明潛在風險）
    5. 建議行動（短期可採行的2項具體行動）

    請確保分析具有洞察力，不要只是重述新聞內容。
    """

    AI分析 = 呼叫LLM(分析提示詞)
    回傳 AI分析
```

## 練習題 / 作業

### 課堂練習

**練習 1：情報需求盤點**

找出在你的工作中，有哪些情報需求是**目前靠人工定期搜集**的：

| 需要搜集的情報 | 目前多久搜集一次 | 需要耗費多少時間 | AI 能幫上的地方 |
|--------------|---------------|---------------|---------------|
| | | | |
| | | | |

**練習 2：Prompt 品質比較**

針對同一組新聞文章，設計兩種不同的分析 Prompt：
- 版本 A：只要求「摘要新聞重點」
- 版本 B：以分析師角度要求「提供洞察與建議」

比較兩個版本的輸出差異，哪個對決策更有幫助？

### 課後作業

1. **情報搜集實作**：選一個你感興趣的產業或公司作為主題，使用 NewsAPI 搜集最近一週的相關新聞（至少 10 篇），然後讓 AI 生成一份簡短的情報摘要（不需要完整研報，500 字以內即可）。

2. **洞察品質評估**：閱讀 AI 生成的摘要，評估：
   - 哪些洞察是你看完新聞也能自己想到的？
   - 哪些洞察是 AI 幫你「串連」出來的、你可能沒注意到的？
   - 如何修改 Prompt 讓洞察更有深度？
