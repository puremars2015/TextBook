# 大成建設 Power Automate 教育訓練課綱（客製版）

目的：8–10 週內培養內部「流程小工廠」，快速落地 6–12 個真正被使用的自動化，降低重複人工、縮短等待與通知延遲，為後續更大範圍的流程治理打基礎。

## 訓練核心原則
1. 先產出可用結果，再補理論原理。
2. 每一堂課結束前必有「部門可部署的最小流程 (MVP)」。
3. 用公司現有資源：SharePoint/OneDrive/Teams/Email/Excel/（公司 ERP / 專案管理系統 / 合約系統）為主。
4. 明確度量：節省工時、減少錯誤、通知提前、資料集中度。
5. 文件化與版本規範從第 3 週開始同步導入。

## 角色分工
- 流程推動者（核心學員 6–10 人）：負責需求拆解與初版流程。
- 部門代表：提出痛點與驗收結果。
- 培訓講師 / 顧問：範例、指導與Review。
- IT / 資訊：帳號、授權、連接器權限、環境隔離。

## 階段與週次概覽
| 階段 | 週 | 主題焦點 | 主要成果 | KPI 範例 |
|------|----|-----------|----------|----------|
| 快速成效 | 1–2 | 雲端觸發 + 通知 + Excel/檔案 | 3 個通知/整理類流程上線 | 減少人工通知 50% |
| 核心技能 | 3–4 | 條件、迴圈、錯誤處理、資料轉換 | 2 個資料彙整流程 | 每日彙整自動化 |
| 部門場景 | 5–7 | 工務/採購/帳務/現場回報整合 | 4–6 個場景流程 | 節省人力 1 人日/週 |

## 週次詳細課綱
### 第 1 週：啟動與三個「立即可做」流程
目標：學員完成首次流量從觸發到通知。
場景示例：
- 現場表單（Microsoft Forms）提交 → Teams 頻道摘要貼文。
- SharePoint 資料夾新增合約 PDF → 寄出審閱提醒。
- Excel 表更新（排程） → 匯出前 5 項逾期項目 email。
交付：3 個雲端流程草稿 + 流程命名規範（前綴、版本）。

### 第 2 週：檔案與資料整理自動化
重點：檔案觸發、條件判斷、分支、動態內容。
場景：
- 每日將各專案 Excel 上傳匯總成一份主控表。
- 上傳照片自動改檔名（專案代碼_日期_流水號）並移動到正確目錄。
- 合約到期前 N 天自動提醒。
交付：1 個合併彙整流程 + 1 個命名/分類流程。

### 第 3 週：控制流程與錯誤處理（加上第一版文件）
重點：Scope、重試、錯誤分支、通知策略、變數與陣列操作。
場景：
- 供應商清單資料清洗：空白欄位補預設、格式轉換。
- 流程失敗集中回報（Teams + Email 群組）。
交付：錯誤處理範本流程 + 流程設計說明模板（Markdown）。

### 第 4 週：資料彙整與報表輸出
重點：Excel / CSV / Dataverse（若有）操作、動態產生報表、排程。
場景：
- 每週工程進度表資料匯總→產出摘要工作表→轉 PDF→寄出。
- 現場安全事件表單→分類統計→儲存與歸檔。
交付：2 個可重複報表流程 + 統一欄位對照表。

### 第 5 週：部門場景 – 採購與合約
重點：多步驟審核、狀態更新、版本迭代。
場景：
- 合約審核階段流轉（建立→法務→主管→歸檔）。
- 採購申請表單→自動建立比價清單→提醒填寫。
交付：1 個多階段審核流程（含狀態欄位）+ 1 個採購起始流程。

### 第 6 週：部門場景 – 工務現場與品質
重點：圖片/附件處理、命名與壓縮、匯總回報。
場景：
- 現場巡檢照片上傳→壓縮→分類→每日合併摘要。
- 缺失整改回報流程（建立→跟催→完成紀錄）。
交付：照片處理子流程 + 巡檢回報主流程。

### 第 7 週：部門場景 – 帳務與發票/進出帳
重點：結構化資料、條件核對、提醒與紀錄追蹤。
場景：
- Excel 應付帳款清單 → 到期提醒與標記。
- 發票掃描 PDF 至資料夾 → OCR 或命名（若需 PAD）→ 登記清單。
交付：到期提醒流程 + 發票歸檔流程（含錯誤通知）。

### 第 8 週：進階整合（Web / 舊系統 / API / PAD）
重點：PAD 錄製、選取器穩定性、重試策略、HTTP 呼叫、跨系統資料搬運。
場景：
- ERP 查詢某專案最新成本 → 取值 → 更新 SharePoint 報表。
- 從外部 API 抓取材料價格 → 比對內部清單 → 產出差異報告。
交付：1 個端到端（API + PAD + 雲端）流程 + PAD 選取器指引。

### 第 9 週：治理與最佳化
重點：命名規範、Solution、環境（Dev/Test/Prod）、重複元件共用、失敗監控儀表板。
交付：治理手冊 V1（命名、目錄、錯誤處理標準）、失敗監控 Flow、流程清單表格。

### 第 10 週：成果驗收與擴展展望
重點：效益盤點、KPI 回顧、遺留缺口、優先級排序、擴展到 Power Apps / BI。
交付：
- 成效報告（節省時間、錯誤率）
- 待改進與下一階段計畫清單
- 流程成熟度評估表

## 每章節模板（實作導向）
1. 背景痛點（3 行內）
2. 目標成果（清晰定義輸入/輸出）
3. 快速路徑（最少步驟達成 MVP）
4. 延伸優化（錯誤處理/彙整/通知分級）
5. 原理拆解（只在延伸後補：觸發器類型、動作限制、最好 vs 次佳方式）
6. 常見失敗模式（選取器不穩、欄位名稱變動、Excel 鎖檔）
7. 驗收指標（什麼算完成）
8. 後續練習（替換資料源、加入審核、改用 Dataverse）

## 快速成果清單（初步目標 12+）
- 表單→Teams 摘要通知
- 合約到期提醒
- 每日照片分類與壓縮歸檔
- Excel 分散檔案→主控表合併
- 供應商清單清洗與轉格式
- 巡檢缺失跟催通知
- 應付帳款到期標記與催辦
- 發票歸檔 + 命名流程
- 材料價格 API 比價報告
- ERP 成本更新自動寫入報表
- 失敗集中告警 Flow
- 周進度報表自動產生與寄送

## 治理與標準（摘要）
- 命名：`[部門]-[場景]-[動詞]-vX`（例：`FIN-AP-Reminder-v1`）
- 變數：語意命名，不用單字母；錯誤變數統一 `ErrorContext`。
- 版本：重大改動升版號，新版並行測試 1 週後替換。
- 權限：敏感連接器需列清單與審批；DLP 原則第 9 週導入。
- 監控：失敗、耗時、重試次數 → 週報表。

## 成功衡量（示例指標）
- 流程部署數：> 12 個
- 平均每流程每月節省人時：> 4 小時
- 失敗率：< 5%（近 30 日）
- 通知延遲：重要事件平均 < 5 分鐘
- 重複手動 Excel 合併次數：下降 80%

## 後續延伸（第二階段建議）
- Power Apps：現場巡檢/缺失提報行動表單
- Power BI：流程成效與錯誤趨勢看板
- AI Builder：發票/合約關鍵欄位擷取
- 自訂連接器：若 ERP/API 需特殊認證
