# Vibe Coding 教學大綱

**副標題：**  
用 AI 把你的工作流程，變成可以執行的程式

---

## 一、課程定位與學習目標

### 課程定位
- 非工程師可上手
- 以 AI 協作開發（Human + AI）為核心
- 解決真實工作問題，而非學語法

### 學習目標
- 能清楚描述工作流程給 AI
- 能讀懂、驗證 AI 產生的程式碼
- 能修改 AI 程式以符合實際需求
- 至少完成 1 個回公司能用的小工具

---

## 二、整體課程架構

| 模組 | 主題 | 核心目的 |
|----|----|----|
| M1 | Vibe Coding 思維 | 建立正確觀念 |
| M2 | AI 協作開發方式 | 學會如何與 AI 協作 |
| M3 | 程式基礎生存包 | 看得懂、不被騙 |
| M4 | 實戰一：流程自動化 | 快速成功 |
| M5 | 除錯與優化 | 工具能實際使用 |
| M6 | 實戰二：個人流程 | 帶回公司用 |

## M1｜什麼是 Vibe Coding

- Vibe Coding 是什麼、不是什麼
- 人與 AI 的分工
- 為什麼現在是最佳時機

**核心觀念**
- 人：意圖、判斷、驗證
- AI：實作、補齊細節、修正

### M1 教學目標（上完這一段學員要能做到）
- 用一句話講清楚什麼是 Vibe Coding（以及它跟「學程式」的差別）
- 知道 AI 在開發裡能做什麼、不能做什麼（避免期待錯誤）
- 會用「意圖 → 產出 → 驗證 → 修正」的迭代思維跟 AI 配合
- 知道一個工具做到什麼程度就算 MVP（可用即可，不追求完美）

---

### M1-1｜Vibe Coding 的定義（講清楚「是什麼」）
**一句話定義：**
> Vibe Coding 是用 AI 當工程助理，把你的工作流程快速翻成「可執行的程式」，你負責方向與驗證，AI 負責實作與補細節。

**課堂要強調的 3 個關鍵字：**
- **工作流程**：先有流程，再有程式（程式只是把流程自動化）
- **可執行**：不是做簡報、不是聊天摘要，是能跑、能產出檔案/結果
- **迭代**：一次到位不重要，能快速修正才重要

---

### M1-2｜Vibe Coding 不是什麼（釐清錯誤期待）
- 不是「不用思考」：你仍需定義目標、判斷對錯、確認結果可用
- 不是「完全不用懂程式」：至少要懂到能讀懂流程、看懂輸入輸出、看懂錯誤類型（M3 會補齊）
- 不是「AI 產的都正確」：AI 會合理化、會猜、會漏條件，必須用驗證把關
- 不是「先追求完美架構」：先做出可用版本，再逐步加規則/例外/效能

---

### M1-3｜人與 AI 的分工（你要教學員「把責任放對位置」）
**人（你/學員）要負責：**
- **意圖與成功標準**：輸出是什麼？要符合哪些規則？錯了算誰錯？
- **資料與例外**：資料來源是什麼？有哪些特殊情境？什麼不能做？
- **驗證與決策**：抽樣檢查、比對、接受/拒絕 AI 的修改

**AI 要負責：**
- 把流程翻成可跑的程式（例如 Python 腳本）
- 補齊細節（資料讀取、格式轉換、迴圈處理、檔案輸出）
- 根據錯誤訊息協助除錯與修正

**一句話提醒（放投影片也好用）：**
> AI 讓「打字寫程式」變便宜，但讓「判斷對不對」更重要。

---

### M1-4｜Vibe Coding 的標準工作迴圈（上課要讓學員形成肌肉記憶）
你可以用 5 步驟口訣教：
1. **講清楚需求**：目標、輸入、輸出、限制、例外
2. **先要最小版本（MVP）**：只做 80% 的主流程
3. **跑起來**：能執行、有結果（即便很醜也可以）
4. **驗證**：用抽樣/對照/檢核規則確認正確性
5. **小步修正**：一次只改一件事，避免越修越亂

---

### M1-5｜什麼叫「做得好」：成功標準與驗證方式
**成功標準（課堂共同語言）：**
- **可重跑**：同樣輸入能得到一致輸出
- **可檢查**：結果能用簡單規則驗證（筆數、欄位、總和、關鍵值）
- **可交付**：同事拿到說明能操作（或至少你自己下週還會用）

**驗證方法（不教語法也能做）：**
- **抽樣比對**：隨機挑 5 筆人工比對
- **總量檢核**：筆數、重複數、缺漏欄位
- **邏輯檢核**：例如日期格式、金額不為負、必填欄位不為空

---

### M1-6｜常見翻車點（新手最容易踩）與對策
- **需求太大** → 先砍到 MVP：只做一張表/一個資料夾/一種格式
- **沒有明確輸入輸出** → 一定要說清楚：檔名、欄位、範例資料、產出位置
- **一次改太多** → 一次只改一段：例如只改讀檔、只改欄位規則
- **只看「看起來對」** → 一定要跑驗證：用規則或抽樣確認

---

### M1-7｜課堂示範（10～15 分鐘，讓學員「感覺到可以成功」）
**示範目的：**用最短路徑讓學員看到：描述流程 → AI 產程式 → 跑出結果 → 驗證 → 小修正。

**示範題材建議（選一個即可）：**
- 批次改檔名（資料夾內檔案加日期/移除空白）
- CSV 欄位整理（只留某些欄位、改欄名、輸出新檔）
- Excel 簡單清理（刪空列、統一日期格式、輸出）

**示範腳本（你可以照這個講）：**
1. 我先講清楚輸入與輸出（秀一個範例檔）
2. 我先要求 AI 做 MVP（只做主流程）
3. 我立刻執行，看到結果
4. 我用 2 個檢核規則驗證
5. 我示範「只改一件事」的修正指令

---

### M1-8｜練習（讓學員把觀念落地）
**練習 1：把工作流程寫成 AI 能懂的需求（5～8 分鐘）**
- 請學員挑一個日常工作（例：整理報表、改檔名、合併資料）
- 用 4 格寫清楚：
	- 輸入：
	- 輸出：
	- 規則：
	- 例外/限制：

**練習 2：把需求縮成 MVP（3～5 分鐘）**
- 問學員：如果只能做 20 分鐘，哪個步驟最值得先自動化？
- 把「非必要」先刪掉，留下主流程

---

### M1-9｜M1 這一段的產出（讓學員帶走可重用的東西）
- 一份「我的流程需求卡」：輸入/輸出/規則/限制（可直接拿去 M2 做 Prompt）
- 一句話定義：我這次要自動化的工作是什麼（用於對齊目標）

> 你也可以把 M1 的作業設定成：每人交一張需求卡，下一模組直接拿來寫 Prompt。

## M3｜基礎程式教學（Vibe Coding 生存包）

### 3.1 程式怎麼讀
- 程式由上往下執行
- 縮排代表區塊
- 看懂每段在做什麼

### 3.2 最低限度邏輯
- if：如果符合條件才做
- for：對一堆資料逐一處理
- function：包起來的步驟

### 3.3 資料結構
- 變數：暫存資料
- 清單 / 陣列：一排資料
- 表格（Excel / CSV）：有欄位的資料

### 3.4 錯誤訊息閱讀
- 環境錯
- 資料錯
- 邏輯錯

### 3.5 套件與環境
- 為什麼要安套件
- pip install 的概念
- 常見環境問題
